# PRD: SpeechLyt — Сервис речевой аналитики для колл-центров

## 1. Обзор продукта

### 1.1 Название
**SpeechLyt** — AI-платформа речевой аналитики для контакт-центров.

### 1.2 Проблема
Колл-центры сталкиваются с критическими проблемами контроля качества:
- Ручная проверка покрывает только 2-5% звонков, остальные 95% остаются без контроля
- Супервизоры тратят 60-70% рабочего времени на прослушивание и оценку звонков
- Отклонения от скриптов и KPI обнаруживаются с задержкой в дни/недели
- Нет объективной картины эмоционального состояния клиентов
- Отсутствует единая система для мониторинга, аналитики и коучинга

### 1.3 Решение
SpeechLyt — облачная платформа, которая автоматически анализирует 100% звонков колл-центра в режиме пост-анализа и реального времени. Платформа проверяет соблюдение скриптов, мониторит KPI, определяет эмоции и генерирует AI-резюме каждого разговора.

### 1.4 Целевая аудитория
- **SMB (5-50 операторов):** небольшие отделы продаж, службы поддержки, медицинские клиники, туристические компании. Нуждаются в простом SaaS с быстрым онбордингом.
- **Enterprise (50-5000+ операторов):** банки, телеком-операторы, страховые компании, ритейл-сети. Требуют кастомизацию, on-premise опцию, SLA, интеграции с корпоративными системами.

### 1.5 Ключевое ценностное предложение
- 100% покрытие звонков вместо 2-5%
- Обнаружение нарушений скриптов в реальном времени
- Снижение AHT на 15-25%, рост конверсии на 10-20%
- Экономия 60-70% времени супервизоров
- Объективная, data-driven оценка качества

## 2. Пользовательские роли

### 2.1 Оператор (Agent)
- Видит свою статистику и оценки
- Получает реал-тайм подсказки во время звонка (Phase 2)
- Просматривает рекомендации по улучшению
- Изучает резюме своих звонков

### 2.2 Супервизор (Supervisor)
- Просматривает дашборд команды с KPI
- Фильтрует звонки по проблемам, оценкам, темам
- Создаёт и редактирует скрипты/чеклисты
- Оставляет комментарии и оценки к звонкам
- Настраивает алерты при критических отклонениях

### 2.3 Руководитель / аналитик (Manager)
- Видит аналитику на уровне всего колл-центра
- Анализирует тренды по периодам
- Экспортирует отчёты
- Управляет KPI-метриками и пороговыми значениями

### 2.4 Администратор (Admin)
- Управляет пользователями и ролями
- Настраивает интеграции (телефония, CRM)
- Конфигурирует ASR и аналитические пайплайны
- Управляет биллингом и тарифами

## 3. Функциональные требования

### Phase 1 — MVP (пост-анализ)

#### 3.1 Загрузка и обработка звонков
- **FR-001:** Система принимает аудиофайлы в форматах WAV, MP3, OGG, FLAC (моно и стерео)
- **FR-002:** Поддержка загрузки через веб-интерфейс (drag & drop), API, и автоматический импорт из интеграций
- **FR-003:** Автоматическое определение и разделение каналов оператор/клиент (speaker diarization)
- **FR-004:** Пакетная загрузка до 10,000 файлов за одну операцию
- **FR-005:** Очередь обработки с приоритизацией и отображением прогресса
- **FR-006:** Хранение оригинальных записей с настраиваемым сроком ретенции (30/60/90/180/365 дней)

#### 3.2 ASR — распознавание речи
- **FR-010:** Транскрибация речи в текст с использованием OpenAI Whisper API
- **FR-011:** Поддержка русского языка как основного, английского как дополнительного
- **FR-012:** Целевая точность распознавания — не менее 90% WER для русского языка на телефонном качестве аудио
- **FR-013:** Временные метки (timestamps) для каждого слова/фразы
- **FR-014:** Возможность подключения альтернативных ASR-провайдеров через абстрактный интерфейс (adapter pattern)
- **FR-015:** Автоматическая пунктуация и нормализация текста

#### 3.3 Проверка скриптов (Script Compliance)
- **FR-020:** Конструктор скриптов — визуальный редактор сценариев разговора с этапами
- **FR-021:** Каждый скрипт состоит из последовательности этапов: приветствие, идентификация, выявление потребностей, презентация, работа с возражениями, закрытие, прощание
- **FR-022:** Для каждого этапа задаются: обязательные фразы/смыслы, запрещённые слова (стоп-слова), максимальное время этапа
- **FR-023:** AI-анализ соответствия — система проверяет не только ключевые слова, но и семантическое соответствие этапу (через LLM)
- **FR-024:** Оценка каждого звонка по скрипту: процент выполнения (0-100%), список пропущенных этапов, список нарушений
- **FR-025:** Гибкие правила: строгое совпадение vs. семантическое соответствие vs. опциональные этапы
- **FR-026:** Поддержка нескольких скриптов (для разных типов звонков: продажи, поддержка, входящие, исходящие)

#### 3.4 KPI-мониторинг
- **FR-030:** Автоматический расчёт метрик из аудио:
  - AHT (Average Handling Time) — средняя длительность разговора
  - Talk/Listen Ratio — соотношение речи оператора и клиента
  - Silence Rate — процент пауз/тишины
  - First Call Resolution (FCR) — определение через AI (решена ли проблема)
  - Hold Time — время удержания клиента
  - Script Compliance Score — оценка соблюдения скрипта
  - Speech Rate (WPM) — скорость речи оператора
- **FR-031:** Кастомные KPI — возможность создавать собственные метрики через конструктор правил
- **FR-032:** Пороговые значения (thresholds) для каждого KPI с цветовой индикацией (зелёный/жёлтый/красный)
- **FR-033:** Дашборд с KPI на уровне: отдельного оператора, команды, всего колл-центра
- **FR-034:** Сравнение KPI по периодам (день, неделя, месяц, квартал)
- **FR-035:** Автоматические алерты при выходе KPI за пороговые значения (email, webhook)

#### 3.5 Эмоциональный анализ
- **FR-040:** Определение эмоционального тона диалога: позитивный, нейтральный, негативный
- **FR-041:** Отдельная эмоциональная оценка для оператора и клиента
- **FR-042:** Детекция критических моментов: конфликт, агрессия, жалоба, благодарность
- **FR-043:** Эмоциональная кривая — визуализация изменения тона на протяжении разговора (timeline)
- **FR-044:** Определение стресс-маркеров в речи оператора
- **FR-045:** Агрегированная эмоциональная статистика по оператору/команде/периоду

#### 3.6 Автоматическое резюме звонков
- **FR-050:** AI-генерация краткого резюме каждого звонка (3-5 предложений) через LLM
- **FR-051:** Структурированное резюме: тема обращения, проблема клиента, решение, итог, следующие шаги
- **FR-052:** Извлечение ключевых сущностей: имена, номера договоров, суммы, даты, продукты
- **FR-053:** Автоматическая категоризация звонков по темам (тематический классификатор)
- **FR-054:** Тегирование звонков: автоматические теги + возможность ручного добавления
- **FR-055:** Поиск по резюме и тегам

#### 3.7 Веб-интерфейс
- **FR-060:** Дашборд — обзорная панель с ключевыми метриками, графиками трендов, алертами
- **FR-061:** Список звонков — таблица с фильтрами, сортировкой, поиском
- **FR-062:** Карточка звонка — полная информация: аудиоплеер с транскрипцией, резюме, оценки скрипта, эмоции, KPI
- **FR-063:** Страница оператора — профиль с персональной статистикой, историей оценок, трендами
- **FR-064:** Конструктор скриптов — визуальный редактор сценариев
- **FR-065:** Настройки KPI — конфигурация метрик и пороговых значений
- **FR-066:** Отчёты — генерация и экспорт отчётов в PDF, Excel
- **FR-067:** Управление пользователями — CRUD для пользователей, ролей, команд
- **FR-068:** Адаптивный дизайн (responsive) для десктопа и планшета

#### 3.8 API и интеграции
- **FR-070:** REST API для всех основных операций (загрузка, получение результатов, управление скриптами)
- **FR-071:** Webhook-уведомления при завершении анализа, нарушениях, алертах
- **FR-072:** Интеграция с популярными IP-телефониями: Asterisk, FreeSWITCH (автоматический импорт записей)
- **FR-073:** Интеграция с CRM: Bitrix24, amoCRM (передача резюме и оценок)
- **FR-074:** SSO-аутентификация (SAML 2.0, OIDC) для enterprise-клиентов
- **FR-075:** API Rate Limiting и документация (OpenAPI/Swagger)

### Phase 2 — Реал-тайм аналитика

#### 3.9 Реал-тайм распознавание и анализ
- **FR-080:** Подключение к аудиопотоку звонка через WebSocket/SIP
- **FR-081:** Стриминговая транскрибация с задержкой не более 2 секунд
- **FR-082:** Реал-тайм проверка скрипта — отображение текущего этапа и отклонений
- **FR-083:** Реал-тайм эмоциональный мониторинг — алерт супервизору при негативном тоне
- **FR-084:** Live-подсказки оператору: рекомендуемые фразы, следующий этап скрипта, ответы на возражения
- **FR-085:** Панель супервизора с live-мониторингом всех активных звонков
- **FR-086:** «Красная кнопка» — алерт супервизору при критическом звонке (конфликт, жалоба на компанию, угроза ухода клиента)

### Phase 3 — Расширенная аналитика

#### 3.10 Продвинутые возможности
- **FR-090:** Предиктивная аналитика — прогнозирование конверсии и CSAT по первым минутам звонка
- **FR-091:** Автоматический коучинг — персонализированные рекомендации оператору на основе паттернов лучших звонков
- **FR-092:** Voice of Customer — агрегированный анализ потребностей и болей клиентов
- **FR-093:** Бенчмаркинг — сравнение метрик с рыночными стандартами
- **FR-094:** A/B тестирование скриптов — сравнение эффективности разных сценариев
- **FR-095:** Мультиязычная поддержка (казахский, узбекский, и др.)
- **FR-096:** Голосовая биометрия — верификация клиента по голосу

## 4. Нефункциональные требования

### 4.1 Производительность
- **NFR-001:** Пост-анализ: обработка 1 минуты аудио за не более 30 секунд
- **NFR-002:** Пакетная обработка: до 100,000 минут аудио в сутки на стартовой инфраструктуре
- **NFR-003:** Реал-тайм: задержка транскрибации не более 2 секунд
- **NFR-004:** Веб-интерфейс: время отклика страниц < 500мс, загрузка дашборда < 2 секунд
- **NFR-005:** API: p99 latency < 200мс для read-операций

### 4.2 Масштабируемость
- **NFR-010:** Горизонтальное масштабирование worker-нод для ASR и аналитики
- **NFR-011:** Поддержка мультитенантной архитектуры (каждый клиент — изолированный tenant)
- **NFR-012:** Автоскейлинг при пиковых нагрузках

### 4.3 Безопасность
- **NFR-020:** Шифрование данных at rest (AES-256) и in transit (TLS 1.3)
- **NFR-021:** RBAC (Role-Based Access Control) с гранулярными правами
- **NFR-022:** Аудит-лог всех действий пользователей
- **NFR-023:** PII-маскировка в транскрипциях (номера карт, паспортные данные, телефоны)
- **NFR-024:** Соответствие 152-ФЗ (персональные данные) для российского рынка
- **NFR-025:** Возможность размещения в российских ЦОД для enterprise-клиентов
- **NFR-026:** Data retention policies с автоматическим удалением по истечении срока

### 4.4 Надёжность
- **NFR-030:** SLA 99.9% uptime для SaaS-версии
- **NFR-031:** Автоматическое восстановление при сбоях ASR-провайдера (retry + fallback)
- **NFR-032:** Бэкапы данных с RPO < 1 час, RTO < 4 часа
- **NFR-033:** Graceful degradation — если ASR недоступен, файлы ставятся в очередь

### 4.5 Наблюдаемость
- **NFR-040:** Структурированное логирование всех компонентов
- **NFR-041:** Метрики: количество обработанных звонков, время обработки, ошибки ASR, использование ресурсов
- **NFR-042:** Алертинг при аномалиях (всплеск ошибок, деградация производительности)
- **NFR-043:** Распределённый трейсинг запросов

## 5. Техническая архитектура (высокоуровневая)

### 5.1 Стек технологий
- **Frontend:** React/Next.js, TypeScript, TailwindCSS, Recharts (графики)
- **Backend:** Python (FastAPI) — основной сервер API
- **ASR Engine:** OpenAI Whisper API (primary), с абстракцией для подключения альтернативных провайдеров
- **LLM:** OpenAI GPT-4o / Claude для семантического анализа, резюме, проверки скриптов
- **Очередь задач:** Celery + Redis (обработка аудио)
- **База данных:** PostgreSQL (основные данные) + ClickHouse (аналитика, метрики)
- **Хранение файлов:** S3-совместимое хранилище (MinIO для on-premise, Yandex Object Storage / AWS S3 для облака)
- **Кэширование:** Redis
- **Поиск:** Elasticsearch (полнотекстовый поиск по транскрипциям)
- **Инфраструктура:** Docker, Kubernetes, Terraform

## 6. Модель данных (основные сущности)

### 6.1 Организация (Organization)
- id, name, plan (free/pro/enterprise), settings, created_at

### 6.2 Пользователь (User)
- id, organization_id, email, name, role (admin/manager/supervisor/agent), team_id

### 6.3 Команда (Team)
- id, organization_id, name, supervisor_id

### 6.4 Скрипт (Script)
- id, organization_id, name, type (sales/support/inbound/outbound), stages[], is_active

### 6.5 Этап скрипта (ScriptStage)
- id, script_id, order, name, required_phrases[], forbidden_words[], is_required, max_duration_seconds

### 6.6 Звонок (Call)
- id, organization_id, agent_id, external_id, audio_url, duration, direction (inbound/outbound), phone_number, created_at, processed_at, status (pending/processing/completed/failed)

### 6.7 Транскрипция (Transcription)
- id, call_id, full_text, segments[] (speaker, text, start_time, end_time, confidence)

### 6.8 Анализ скрипта (ScriptAnalysis)
- id, call_id, script_id, overall_score, stage_results[] (stage_id, status, details), violations[]

### 6.9 Эмоциональный анализ (EmotionAnalysis)
- id, call_id, overall_sentiment (positive/neutral/negative), agent_sentiment, client_sentiment, emotion_timeline[], critical_moments[]

### 6.10 KPI-результат (CallKPI)
- id, call_id, metrics (aht, talk_listen_ratio, silence_rate, speech_rate, hold_time, script_score, fcr)

### 6.11 Резюме (CallSummary)
- id, call_id, short_summary, topic, problem, solution, outcome, next_steps, entities[], tags[]

### 6.12 Алерт (Alert)
- id, organization_id, call_id, type, severity (info/warning/critical), message, acknowledged

## 7. Тарифные планы

### 7.1 Starter (SMB)
- До 1,000 минут аудио/месяц
- До 10 операторов
- Пост-анализ
- Базовые скрипты (до 5)
- Email-поддержка
- **Цена:** ~15,000 ₽/мес

### 7.2 Professional (Mid-market)
- До 10,000 минут аудио/месяц
- До 50 операторов
- Пост-анализ + реал-тайм (Phase 2)
- Неограниченные скрипты
- CRM-интеграции
- Приоритетная поддержка
- **Цена:** ~60,000 ₽/мес

### 7.3 Enterprise
- Неограниченные минуты
- Неограниченные операторы
- Все функции + on-premise опция
- SSO, кастомные интеграции

## 9. Роадмап

### Phase 1 — MVP (3 месяца)
- Загрузка и обработка аудио
- ASR через OpenAI Whisper API
- Speaker diarization
- Проверка скриптов (конструктор + AI-анализ)
- KPI-мониторинг с дашбордами
- Эмоциональный анализ (пост-обработка)
- AI-резюме звонков
- Веб-интерфейс (дашборд, список звонков, карточка звонка, скрипты, KPI)
- REST API + webhook-уведомления
- Авторизация и мультитенантность
- Интеграция с Asterisk (автоимпорт записей)

### Phase 2 — Реал-тайм + интеграции (3 месяца)
- Стриминговая транскрибация через WebSocket
- Реал-тайм подсказки оператору
- Live-мониторинг для супервизора
- Интеграция с Bitrix24, amoCRM
- SSO (SAML 2.0, OIDC)
- Мобильный адаптивный интерфейс
- Расширенная отчётность (PDF, Excel)

### Phase 3 — Advanced Analytics (3 месяца)
- Предиктивная аналитика
- Автоматический коучинг
- Voice of Customer
- A/B тестирование скриптов
- Голосовая биометрия
- Мультиязычная поддержка (казахский, узбекский)
- On-premise deployment пакет

## 10. Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Низкое качество ASR на телефонном аудио | Высокая | Высокое | Предобработка аудио (noise reduction), fallback на альтернативный ASR, тюнинг параметров Whisper |
| Высокая стоимость API-вызовов (OpenAI) при масштабировании | Средняя | Высокое | Кэширование, батчинг, переход на self-hosted Whisper при росте, оптимизация промптов для LLM |
| Задержка реал-тайм обработки > 2 сек | Средняя | Среднее | Стриминговый ASR, оптимизация пайплайна, edge-обработка |
| Конкуренция с зрелыми платформами (Yandex, T-Bank) | Высокая | Среднее | Фокус на UX, быстрый онбординг для SMB, гибкие скрипты, конкурентная цена |
| 152-ФЗ — требования по хранению ПД | Средняя | Высокое | Размещение в российских ЦОД, PII-маскировка, on-premise опция |
| Зависимость от одного ASR-провайдера | Средняя | Среднее | Абстрактный интерфейс ASR, готовность подключить Yandex SpeechKit / self-hosted Whisper |

## 11. Ограничения и допущения

### Допущения
- Клиенты предоставляют аудио в приемлемом качестве (битрейт >= 8kHz, SNR > 10dB)
- Для MVP достаточно поддержки русского и английского языков
- OpenAI Whisper API обеспечивает достаточную точность для телефонного аудио на русском языке
- Основной сценарий использования — двусторонний диалог (оператор + клиент)

### Ограничения MVP
- Реал-тайм аналитика не входит в MVP (Phase 2)
- On-premise вариант не входит в MVP (Phase 3)
- Биометрия голоса не входит в MVP (Phase 3)
- MVP поддерживает только русский и английский языки
